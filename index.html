Lab 1: Study-Room Booking System (SRBS) Paper Prototype

Write a few sentences about yourself, your class project, or your website idea.

The system addresses study room booking issues where double bookings, cancellations, and no-shows prevent students from seeing accurate availability and accessing rooms fairly. It provides real-time booking visibility, prevents duplicate reservations per student, sends reminders before bookings, and requires confirmation to reduce no-shows and cancellations. It supports operations through student verification, decision-making through automatic cancellation of unconfirmed bookings, and customer support through clear booking rules and notifications. The system involves students and staff, uses a booking website, follows a structured reservation and confirmation process, and manages booking data. Currently, the process leads to about 50% of booked hours being wasted due to cancellations or no-shows (e.g., Room R1). To improve efficiency, the system enforces confirmation deadlines so unconfirmed reservations are automatically cancelled to free up rooms for other students. Class link

Lab 3: Smart Campus SRBS ICT + EA Plan

This assignment asks you to design a Smart Campus Study Room Booking system and present it as an ICT plan combined with an Enterprise Architecture (EA) blueprint. The goal is to show how user needs, technology, network infrastructure, and system design all work together to create an efficient booking solution. Your submission should include user stories, hardware and software components, a network plan with security considerations, and an EA blueprint covering business, data, application, and technology layers. You’ll also briefly explain how packet switching and TCP/IP protocols support the system, and finish with a one-page recommendation summarizing your overall solution.

Lab 4: Student Information System (SIS) - database

This assignment requires you to design a simple Student Information System (SIS) database using a web portal and SQLite. The system must manage students, faculty, courses, enrollments, and grades while supporting key processes like enrolling, dropping courses, and submitting grades. You will create tables with primary and foreign keys, handle many-to-many relationships using an enrollment (bridge) table, and enforce rules such as preventing duplicate enrollments. The system should also generate basic reports like course seat availability and student transcripts.
